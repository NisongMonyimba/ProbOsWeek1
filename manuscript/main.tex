\section{Abstract}
% ============================================================

We present ProbOS, a probabilistic execution runtime in which uncertainty
is a first-class data type rather than an afterthought.
The system is structured as a layered kernel analogous to a classical
operating system: distributions form the type system (Week~1),
domain models form the process abstraction (Week~2),
a Monte Carlo engine forms the scheduler (Week~3),
and a causal provenance graph forms the audit trail (Week~3).

This paper documents Weeks~1 through~3.
Week~1 delivers a \texttt{Distribution} abstract base class (ABC) with
five concrete implementations, verified by 44~Python and 13~C++ tests.
Week~2 delivers a \texttt{Model} ABC and \texttt{BatteryModel2Cell},
a vectorised 8-state Arrhenius ODE with 15~uncertain parameters,
validated against Kim et al.~\cite{kim2007} accelerating rate calorimetry.
Week~3 delivers the \texttt{MonteCarloEngine} producing P05/P50/P95
trajectories with convergence certificates, \texttt{SobolSensitivity}
identifying dominant parameters via first-order and total-effect indices,
and \texttt{ProvenanceTracker} recording which parameter combinations
drove the dangerous tail outcomes.

All code passes \texttt{mypy} strict type checking and \texttt{ruff} lint
with zero errors across 202~Python tests and 13~C++ tests.
The full codebase is publicly available at
\url{https://github.com/NisongMonyimba/ProbOs}.

\section{Introduction}

\subsection{Motivation}

Deterministic simulation uses a single parameter value where there should
be a distribution.
A deterministic battery model uses one activation energy.
ProbOS simulates 5\,000 batteries simultaneously and finds that the P95
worst-case cell (hottest 5\%) heats significantly faster than the median,
and that activation energy $E_{a,\text{SEI}}$ alone explains 45.7\% of
the temperature variance (first-order Sobol index $S_1 = 0.457$).
That tail risk and its causal origin are completely invisible to a
deterministic model.

\section{Mathematical Background}
% ============================================================

\subsection{Probability Distributions and Log-PDF Stability}

\begin{proposition}[Log-PDF stability]
\label{prop:logpdf}
The analytical log-density of $\mathcal{N}(\mu, \sigma^2)$,
\[
  \ell(x) = -\tfrac{1}{2}\log(2\pi\sigma^2)
            - \frac{(x-\mu)^2}{2\sigma^2},
\]
is finite for all $x \in \mathbb{R}$ and all $\sigma > 0$.
The numerical approximation $\hat\ell(x) = \log(\hat f(x))$ satisfies
$\hat\ell(x) = -\infty$ whenever $|x-\mu|/\sigma \gtrsim 37.5$
due to IEEE~754 double-precision underflow.
\end{proposition}

\subsection{The Arrhenius Rate Law}

\begin{equation}
  k(T) = A \exp\!\left(-\frac{E_a}{RT}\right)
  \label{eq:arrhenius}
\end{equation}

\subsection{Monte Carlo Estimation and the CLT}

Given $N$ i.i.d.\ samples $x_1,\ldots,x_N \sim f$,
the Monte Carlo estimator $\hat\mu_N = N^{-1}\sum x_i$ satisfies:
\begin{equation}
  \mathbb{E}[|\hat\mu_N - \mu|] \approx \frac{\sigma}{\sqrt{N}}
  \label{eq:clt}
\end{equation}
where $\sigma^2 = \mathrm{Var}[X]$.
We verify this empirically in Section~\ref{sec:clt}.

\subsection{Sobol Sensitivity Indices}

The ANOVA-style variance decomposition is:
\begin{equation}
  V(Y) = \sum_i V_i + \sum_{i<j} V_{ij} + \cdots
\end{equation}
The first-order index $S_{1,i} = V_i / V(Y)$ measures the direct
contribution of parameter $i$ alone.
The total-effect index $S_{T,i} = 1 - V_{\sim i}/V(Y)$ measures all
contributions involving parameter $i$, including interactions.
By construction, $S_{1,i} \le S_{T,i}$ and $\sum_i S_{1,i} \le 1$.

\section{System Architecture}
% ============================================================

\subsection{Design Principles}

\begin{enumerate}
  \item \textbf{Uncertainty by default.}
    Every computation accepts and returns distributions.
  \item \textbf{Analytical log-density.}
    Every distribution implements \texttt{log\_pdf} analytically.
  \item \textbf{Vectorised batch computation.}
    No Python for-loops over particles; NumPy broadcasting only.
  \item \textbf{Physical bounds enforcement.}
    Models clip state variables to physically valid ranges.
  \item \textbf{Strict type safety.}
    All code passes \texttt{mypy --strict} with zero errors.
  \item \textbf{Causal provenance.}
    Every output is traceable to the parameter combination that caused it.
\end{enumerate}

\subsection{Layer Structure}

\begin{description}
  \item[Layer 1 -- Type System (Week 1):]
    \texttt{Distribution} ABC and five concrete implementations.
  \item[Layer 2 -- Model Layer (Week 2):]
    \texttt{Model} ABC and \texttt{BatteryModel2Cell}.
  \item[Layer 3 -- Monte Carlo Engine (Week 3):]
    \texttt{MonteCarloEngine}, \texttt{SobolSensitivity},
    \texttt{ProvenanceTracker}.
\end{description}

\section{Week 2: Model Layer}
% ============================================================

\subsection{The Model ABC}

The \texttt{Model} ABC enforces the interface that every domain model
must implement:
\texttt{state\_dim}, \texttt{param\_dim}, \texttt{param\_names()},
\texttt{initial\_state()}, and \texttt{forward\_batch(state, params, dt)}.
The key constraint is that \texttt{forward\_batch} must advance all
$N$ particles simultaneously using NumPy broadcasting.

\subsection{BatteryModel2Cell}

The three sequential exothermic reactions follow the Arrhenius law
(Equation~\ref{eq:arrhenius}):
\begin{align}
  \frac{dc_r}{dt} &= -k_r(T)\, c_r \\
  Q_r &= H_r \cdot m_\text{cell} \cdot \left(-\frac{dc_r}{dt}\right) \\
  \frac{dT}{dt} &= \frac{\sum_r Q_r - hA(T - T_\text{amb})}
                        {m_\text{cell} C_p}
\end{align}

The model has \texttt{state\_dim}=8 and \texttt{param\_dim}=15,
with nominal parameters from Kim et al.~\cite{kim2007}.

\section{Week 3: Monte Carlo Engine}
% ============================================================

\subsection{MonteCarloEngine}

\begin{algorithm}[h]
\caption{MonteCarloEngine.run()}
\begin{algorithmic}[1]
\State Draw params $\in \mathbb{R}^{N \times d}$ from priors
\State Tile initial\_state to shape $(N, s)$
\State Store state at $t=0$ in trajectories$[:,0,:]$
\For{$t = 1$ \textbf{to} $n\_steps$}
  \State state $\leftarrow$ model.forward\_batch(state, params, dt)
  \State trajectories$[:,t,:] \leftarrow$ state
\EndFor
\State percentiles $\leftarrow$ np.percentile(trajectories, [5,50,95], axis=0)
\State convergence $\leftarrow \sigma(\text{final\_state}) / \sqrt{N}$
\State \Return MCResult
\end{algorithmic}
\end{algorithm}

The inner loop iterates over time steps, not particles.
All $N$ particles are advanced simultaneously in each
\texttt{forward\_batch} call.

\subsection{Convergence Certificate}

By the CLT (Equation~\ref{eq:clt}), the 95\% confidence half-width
on the mean trajectory is $1.96 \cdot \sigma/\sqrt{N}$.
At $N=5000$ for $T_1$: $\sigma/\sqrt{N} = 7.30$~K,
giving a 95\% CI of $\pm 14.3$~K.

\subsection{Causal Provenance Graph}

The \texttt{ProvenanceTracker} records a directed acyclic graph (DAG)
with two node types:
\begin{itemize}
  \item \textbf{Param nodes}: one per (particle, parameter), timeless.
  \item \textbf{State nodes}: one per (particle, state variable, timestep),
    with all param nodes as parents.
\end{itemize}

The P05 query identifies which particles lie in the dangerous tail:
\begin{equation}
  \text{P05 particles} = \{i : x_i^{(T_1)}[t_\text{final}]
  \le \text{percentile}(X^{(T_1)}, 5\%)\}
\end{equation}

For $N=200$ particles, approximately 5\% ($\approx$10) lie in the P05
tail. Their ancestor param nodes reveal that the P05 tail has higher
mean $E_{a,\text{SEI}}$ than the overall population
($144\,354$ vs $135\,080$~J~mol$^{-1}$) -- consistent with higher
activation energy producing slower SEI decomposition, lower heat
release, and consequently lower temperature at short timescales.

\section{Conclusion}
% ============================================================

We have presented Weeks~1--3 of ProbOS, a probabilistic execution
runtime treating uncertainty as a first-class data type.

Week~1 established the type system: a \texttt{Distribution} ABC
enforcing analytical log-density, five concrete distributions, and
44~Python + 13~C++ tests.

Week~2 established the model layer: a \texttt{Model} ABC enforcing
vectorised batch computation, \texttt{BatteryModel2Cell} implementing
an 8-state Arrhenius ODE with 15~uncertain parameters, and 67~new tests
including Kim~2007 ARC validation.

Week~3 established the Monte Carlo engine: \texttt{MonteCarloEngine}
producing P05/P50/P95 trajectories with $\sigma/\sqrt{N}$ convergence
certificates; \texttt{SobolSensitivity} revealing that $E_{a,\text{SEI}}$
explains 45.7\% of thermal runaway variance ($S_1 = 0.457$);
and \texttt{ProvenanceTracker} providing regulatory-grade causal
audit trails linking dangerous tail outcomes to their generating
parameter combinations.

The combined codebase comprises 202~Python tests and 13~C++ tests,
passing \texttt{mypy} strict and \texttt{ruff} with zero errors.
